# Инструкция по обновлению ИИ-ассистента техподдержки

## 📖 О документе

Это руководство описывает процессы обновления базы знаний, системного промпта и конфигурации ИИ-ассистента.

---

## 1️⃣ Обновление базы знаний

### Форматы базы знаний

База знаний поддерживается в следующих форматах:

| Формат | Файл | Описание |
|--------|------|----------|
| **CSV** | `knowledge_base/knowledge_base.csv` | Основной формат, используется для RAG-поиска |
| **PDF** | `knowledge_base/knowledge_base.pdf` | Для документации и руководств |
| **Markdown** | `docs/*.md` | Для системных промптов и инструкций |

### Структура CSV-файла

```csv
id,title,content,type,tags,source
faq_001,Как создать аккаунт?,Чтобы создать аккаунт...,faq,"регистрация,аккаунт",kb_seed
```

**Поля:**
- `id` — уникальный идентификатор (префикс указывает тип: faq_, doc_, tpl_, scenario_, policy_)
- `title` — заголовок статьи/вопроса
- `content` — содержимое (ответ, инструкция, шаблон)
- `type` — тип записи (faq, document, template, scenario, policy, escalation, integration)
- `tags` — теги через запятую для поиска
- `source` — источник записи

### Добавление новой записи

**Шаг 1:** Откройте `knowledge_base/knowledge_base.csv`

**Шаг 2:** Добавьте новую строку:
```csv
faq_021,Новый вопрос,Ответ на новый вопрос...,faq,"тег1,тег2",kb_seed
```

**Шаг 3:** Пересоберите базу знаний:
```bash
python -m app.tools.build_kb
```

**Шаг 4:** Переиндексируйте в ChromaDB (если используется):
```bash
python -m app.tools.ingest_kb
```

### Массовое обновление из Google Docs

**Шаг 1:** Экспортируйте Google Docs в CSV

1. Откройте Google Doc с базой знаний
2. Файл → Скачать → CSV

**Шаг 2:** Приведите к нужному формату

Убедитесь, что столбцы соответствуют структуре:
```
id,title,content,type,tags,source
```

**Шаг 3:** Замените файл и переиндексируйте

```bash
# Копируйте новый файл
copy path/to/exported.csv knowledge_base/knowledge_base.csv

# Пересоберите
python -m app.tools.build_kb

# Переиндексируйте
python -m app.tools.ingest_kb
```

### Загрузка PDF-документов

Для больших документов (инструкции, руководства):

**Шаг 1:** Поместите PDF в папку `knowledge_base/`

**Шаг 2:** Загрузите в ChromaDB:
```bash
# Один файл
python -m app.tools.ingest_pdf ./knowledge_base/manual.pdf

# Все PDF в папке
python -m app.tools.ingest_pdf ./knowledge_base/ --all
```

**Параметры чанкирования (опционально):**

Измените в `app/tools/pdf_parser.py`:
```python
CHUNK_SIZE = 1000  # символов в чанке
CHUNK_OVERLAP = 100  # перекрытие между чанками
```

## 2️⃣ Обновление системного промпта

### Расположение

Системный промпт находится в файле `docs/system_prompt.md`

### Внесение изменений

**Шаг 1:** Откройте `docs/system_prompt.md`

**Шаг 2:** Отредактируйте нужные разделы:
- Цель ассистента
- Стиль общения
- Сценарии взаимодействия
- Ограничения
- Примеры ответов

**Шаг 3:** Обновите версию в конце файла:
```markdown
**Версия промпта:** 2.0  
**Дата обновления:** 2026-04-17
```

**Шаг 4:** Перезапустите бота/приложение

Промпт загружается при старте, поэтому требуется перезапуск.

### Тестирование нового промпта

**Шаг 1:** Запустите тестовые запросы:
```bash
python -m app.tools.test_prompt --query "Как создать аккаунт?"
```

**Шаг 2:** Проверьте ответы на соответствие:
- Тону бренда
- Полноте информации
- Наличию ссылок на источники
- Корректности ограничений

## 3️⃣ Обновление шаблонов ответов

### Расположение шаблонов

Шаблоны хранятся в базе знаний с типом `template`:
```csv
tpl_001,Шаблон: приветствие нового клиента,Уважаемый(ая) {Имя}!...,template,"приветствие",kb_seed
```

### Переменные в шаблонах

Используйте переменные в фигурных скобках:
- `{Имя}` — имя клиента
- `{Компания}` — название компании
- `{Номер}` — номер заказа
- `{Дата}` — дата
- `{Ссылка}` — URL
- `{Товары}` — список товаров

### Добавление нового шаблона

**Шаг 1:** Определите назначение шаблона

**Шаг 2:** Добавьте в CSV:
```csv
tpl_009,Шаблон: новый сценарий,Текст шаблона с {Переменные},template,"тег1,тег2",kb_seed
```

**Шаг 3:** Обновите базу:
```bash
python -m app.tools.build_kb
```

## 4️⃣ Обновление сценариев обработки

### Типы сценариев

| Тип | Префикс ID | Описание |
|-----|------------|----------|
| FAQ | `faq_XXX` | Часто задаваемые вопросы |
| Документ | `doc_XXX` | Политики, регламенты |
| Шаблон | `tpl_XXX` | Шаблоны ответов |
| Сценарий | `scenario_XXX` | Алгоритмы обработки |
| Политика | `policy_XXX` | Правила и условия |
| Эскалация | `escalation_XXX` | Процедуры эскалации |

### Изменение сценария

**Пример:** Обновление сценария возврата

**Было:**
```csv
scenario_001,Сценарий: возврат,1) Узнать причину...,scenario,"возврат",kb_seed
```

**Стало:**
```csv
scenario_001,Сценарий: возврат,1) Узнать причину. 2) Проверить срок. 3) Запросить фото. 4) Создать заявку. 5) Отправить инструкцию. 6) Контролировать.,scenario,"возврат,обновлено",kb_seed
```

## 5️⃣ Обновление конфигурации

### Переменные окружения (.env)

```bash
# API настройки
OPENAI_API_KEY=your_api_key
OPENAI_BASE_URL=https://api.proxyapi.ru/openai/v1
OPENAI_MODEL=gpt-4.1-mini
OPENAI_EMBEDDING_MODEL=text-embedding-3-small

# RAG настройки
RAG_MODE=auto
CHROMA_PATH=./data/chroma
CHROMA_COLLECTION=customer_feedback_kb

# Telegram (если используется бот)
TELEGRAM_BOT_TOKEN=your_bot_token

# Хранение
SQLITE_PATH=./data/feedback.sqlite

# Логирование
LOG_LEVEL=INFO
TZ=Europe/Moscow
```

### Смена модели

**Шаг 1:** Измените в `.env`:
```bash
OPENAI_MODEL=gpt-4o-mini
```

**Шаг 2:** При смене embedding модели переиндексируйте ChromaDB:
```bash
# Если сменили OPENAI_EMBEDDING_MODEL
python -m app.tools.ingest_kb
```

## 7️⃣ Очистка и сброс данных

### Очистка SQLite

```bash
# Удалить базу данных
rm data/feedback.sqlite

# Или через SQL
sqlite3 data/feedback.sqlite "DELETE FROM feedback;"
```

### Очистка ChromaDB

```bash
# Удалить коллекцию
rm -rf data/chroma

# При следующем запуске создастся заново
python -m app.tools.ingest_kb
```

### Полный сброс

```bash
# Windows PowerShell
Remove-Item -Recurse -Force data/
python -m app.tools.build_kb
python -m app.tools.ingest_kb
```

## 8️⃣ Версионирование и откат

### Git для версионирования

**Коммит изменений базы знаний:**
```bash
git add knowledge_base/knowledge_base.csv
git commit -m "feat: добавлены 10 новых FAQ по оплате"
git push origin main
```

**Откат к предыдущей версии:**
```bash
# Просмотр истории
git log -- knowledge_base/knowledge_base.csv

# Откат к коммиту
git checkout <commit-hash> -- knowledge_base/knowledge_base.csv

# Переиндексация
python -m app.tools.ingest_kb
```

### Бэкап базы знаний

**Создание бэкапа:**
```bash
# PowerShell
$timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
Copy-Item knowledge_base/knowledge_base.csv "backups/kb_backup_$timestamp.csv"
```

**Восстановление из бэкапа:**
```bash
Copy-Item backups/kb_backup_20260417_120000.csv knowledge_base/knowledge_base.csv
python -m app.tools.ingest_kb
```

## 9️⃣ Мониторинг качества

### Метрики для отслеживания

| Метрика | Цель | Как измерить |
|---------|------|--------------|
| % решённых без эскалации | >80% | Аналитика за период |
| Среднее время ответа | <30 сек | Логи системы |
| CSAT (удовлетворённость) | >4.5/5 | Опросы после диалога |
| Точность RAG-поиска | >90% | Тестовые запросы |

### Еженедельная проверка

**Чек-лист:**

- [ ] Проверить топ-10 вопросов без ответа
- [ ] Проанализировать эскалации
- [ ] Обновить устаревшие FAQ
- [ ] Добавить новые шаблоны
- [ ] Проверить актуальность ссылок
